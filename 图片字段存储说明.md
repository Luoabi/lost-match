# 图片字段存储说明

## 正确的存储方式

### 后端返回格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "url": "http://localhost:8080/api/uploads/2024/12/20/abc123.jpg",
    "relativePath": "/api/uploads/2024/12/20/abc123.jpg",
    "filename": "avatar.jpg",
    "size": 102400
  }
}
```

### 前端存储方式

#### ✅ 正确做法：存储完整URL
```javascript
const res = await uploadFile(file.raw)
if (res.code === 200) {
  // 直接使用 data.url（完整URL）
  profileForm.value.avatar = res.data.url
  // 结果：avatar = "http://localhost:8080/api/uploads/2024/12/20/abc123.jpg"
}
```

#### ❌ 错误做法：只存储部分路径
```javascript
// 错误1：只存储 data（整个对象）
profileForm.value.avatar = res.data
// 结果：avatar = { url: "...", relativePath: "...", ... }

// 错误2：只存储 relativePath
profileForm.value.avatar = res.data.relativePath
// 结果：avatar = "/api/uploads/2024/12/20/abc123.jpg"
// 问题：前端显示时需要手动拼接域名
```

## 完整流程示例

### 1. 用户上传头像
```javascript
const handleAvatarChange = async (file) => {
  // 验证文件
  const isImage = file.raw.type.startsWith('image/')
  const isLt2M = file.raw.size / 1024 / 1024 < 2
  
  if (!isImage) {
    ElMessage.error('只能上传图片文件!')
    return
  }
  if (!isLt2M) {
    ElMessage.error('图片大小不能超过 2MB!')
    return
  }

  try {
    // 上传文件
    const { uploadFile } = await import('@/api/file')
    const res = await uploadFile(file.raw)
    
    if (res.code === 200) {
      // ✅ 存储完整URL到avatar字段
      profileForm.value.avatar = res.data.url
      ElMessage.success('头像上传成功')
    }
  } catch (error) {
    ElMessage.error('头像上传失败')
  }
}
```

### 2. 保存到数据库
```javascript
const handleSaveProfile = async () => {
  try {
    await updateUserProfile({
      id: userInfo.value.id,
      username: profileForm.value.username,
      email: profileForm.value.email,
      phone: profileForm.value.phone,
      avatar: profileForm.value.avatar  // 完整URL
    })
    
    ElMessage.success('个人信息更新成功')
  } catch (error) {
    ElMessage.error('更新失败')
  }
}
```

### 3. 后端保存
```java
@Transactional
public User updateProfile(User user) {
    User existingUser = userRepository.findById(user.getId())
            .orElseThrow(() -> new RuntimeException("用户不存在"));
    
    if (user.getAvatar() != null) {
        // 直接保存完整URL
        existingUser.setAvatar(user.getAvatar());
    }
    
    return userRepository.save(existingUser);
}
```

## 优势总结

### 存储完整URL的优势
1. ✅ **前端使用简单**：直接绑定，无需处理
2. ✅ **跨域友好**：支持CDN、对象存储等外部URL
3. ✅ **迁移方便**：更换域名只需更新数据库
4. ✅ **调试容易**：可以直接复制URL在浏览器测试
5. ✅ **兼容性好**：支持http、https、相对路径等多种格式

### 注意事项
1. 数据库字段长度要足够（建议VARCHAR(500)）
2. 如果更换域名，需要批量更新数据库中的URL
3. 生产环境建议使用CDN或对象存储
