# 微信小程序图片显示问题解决方案

## 问题原因

1. **相对路径问题**：后端返回的图片 URL 是相对路径 `/api/uploads/2026/04/12/xxx.png`，小程序无法直接访问
2. **域名白名单**：微信小程序要求所有网络图片必须配置域名白名单

## 解决步骤

### 1. 使用图片 URL 处理工具

已创建 `utils/image.js` 工具函数，用于将相对路径转换为完整 URL。

**使用方法：**

```javascript
const { getImageUrl, getImageUrls } = require('../../utils/image.js');

// 单个图片
const fullUrl = getImageUrl('/api/uploads/2026/04/12/xxx.png');
// 结果：http://localhost:8080/api/uploads/2026/04/12/xxx.png

// 批量处理
const images = ['/api/uploads/...', '/api/uploads/...'];
const fullUrls = getImageUrls(images);
```

### 2. 在页面中使用

**示例：失物列表页面**

```javascript
const { getLostItemList } = require('../../../api/lostItem.js');
const { getImageUrl } = require('../../../utils/image.js');

// 在获取数据后处理图片 URL
const newList = res.data.list.map(item => ({
  ...item,
  images: item.images && item.images.length > 0 
    ? item.images.map(url => getImageUrl(url))
    : []
}));
```

### 4. 配置微信小程序域名白名单

**开发阶段（本地测试）：**

1. 打开微信开发者工具
2. 点击右上角"详情"
3. 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

**生产环境：**

1. 登录微信公众平台：https://mp.weixin.qq.com
2. 进入"开发" -> "开发管理" -> "开发设置"
3. 找到"服务器域名"
4. 在"uploadFile合法域名"和"downloadFile合法域名"中添加：
   - `https://你的域名.com`（注意：必须是 HTTPS）

### 5. 后端配置静态资源访问

确保后端已配置静态资源映射，让 `/api/uploads/` 路径可以访问到文件。

**Spring Boot 配置（WebConfig.java）：**

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/api/uploads/**")
                .addResourceLocations("file:uploads/");
    }
}
```

## 快速测试

1. 上传一张图片，获取返回的 URL
2. 在浏览器中访问完整 URL：`http://localhost:8080/api/uploads/2026/04/12/xxx.png`
3. 如果能看到图片，说明后端配置正确
4. 在小程序中使用 `getImageUrl()` 处理后显示

## 常见问题

### Q1: 图片还是不显示？
- 检查控制台是否有 404 错误
- 检查图片 URL 是否正确拼接
- 检查后端静态资源配置

### Q2: 真机预览图片不显示？
- 必须配置域名白名单
- 域名必须是 HTTPS
- 或者在开发者工具中勾选"不校验合法域名"

### Q3: 图片上传成功但列表不显示？
- 检查是否使用了 `getImageUrl()` 处理
- 检查数据库中保存的 URL 格式
- 使用 `console.log` 打印处理后的 URL

## 注意事项

1. **开发环境**：使用 `http://localhost:8080`
2. **生产环境**：需要改为 `https://你的域名.com`
3. **配置文件**：在 `config/index.ts` 中统一管理域名配置
4. **图片大小**：建议压缩图片，单张不超过 5MB
