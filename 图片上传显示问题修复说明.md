# 图片上传显示问题修复说明

## 问题描述
图片上传到后端 `uploads` 目录后，前端无法正常显示图片。

## 问题原因
1. 后端返回的是相对路径 `/api/uploads/...`
2. 前端需要完整的URL才能正确显示图片
3. 静态资源配置可能不完整

## 解决方案

### 后端修改

#### 1. FileController.java
**修改内容：**
- 修改返回类型从 `String` 改为 `FileUploadResponse` 对象
- 返回完整的URL（包含域名和端口）
- 添加 `getBaseUrl()` 方法自动获取服务器地址
- 添加 `FileUploadResponse` 响应类

**返回数据结构：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "url": "http://localhost:8080/api/uploads/2024/12/20/xxx.jpg",
    "relativePath": "/api/uploads/2024/12/20/xxx.jpg",
    "filename": "avatar.jpg",
    "size": 102400
  }
}
```

#### 2. WebConfig.java
**优化内容：**
- 添加详细的启动日志，方便调试
- 设置静态资源缓存时间（1小时）
- 添加跨域配置作为备用

### 访问路径说明

**文件存储位置：**
```
项目根目录/uploads/2024/12/20/xxx.jpg
```

**访问URL：**
```
http://localhost:8080/api/uploads/2024/12/20/xxx.jpg
```

**路径映射关系：**
- 请求路径：`/api/uploads/**`
- 映射到：`file:项目根目录/uploads/`
- 由于 context-path 是 `/api`，所以完整路径是 `/api/uploads/**`

## 常见问题排查

### 问题1：404 Not Found
**可能原因：**
- uploads 目录不存在
- 文件路径不正确
- 静态资源配置未生效

**解决方法：**
1. 检查项目根目录是否有 `uploads` 文件夹
2. 检查文件是否真的保存在该目录
3. 重启后端服务，确保配置生效

### 问题2：图片显示不出来
**可能原因：**
- 跨域问题
- URL不正确
- 文件权限问题

**解决方法：**
1. 打开浏览器开发者工具，查看 Network 标签
2. 检查图片请求的状态码和响应
3. 确认返回的URL是完整的（包含 http://）

### 问题3：本地可以，部署后不行
**可能原因：**
- 服务器地址配置问题
- 防火墙或安全组限制

**解决方法：**
1. 确保 `getBaseUrl()` 方法能正确获取服务器地址
2. 如果使用反向代理（如 Nginx），需要配置正确的 proxy_pass
3. 检查服务器防火墙是否开放了相应端口

## 前端使用示例

### 个人中心头像上传
```javascript
const handleAvatarChange = async (file) => {
  try {
    const res = await uploadFile(file.raw)
    if (res.code === 200) {
      // 使用返回的完整URL
      profileForm.value.avatar = res.data.url
      ElMessage.success('头像上传成功')
    }
  } catch (error) {
    ElMessage.error('头像上传失败')
  }
}
```

### 显示头像
```vue
<template>
  <img :src="profileForm.avatar" class="avatar-preview" />
</template>
```

## 生产环境建议

1. **使用CDN或对象存储**
   - 阿里云OSS
   - 腾讯云COS
   - 七牛云
   - AWS S3

2. **配置Nginx反向代理**
   ```nginx
   location /api/uploads/ {
       alias /path/to/uploads/;
       expires 30d;
       access_log off;
   }
   ```

3. **图片压缩和优化**
   - 上传时自动压缩
   - 生成多种尺寸的缩略图
   - 使用WebP格式

4. **安全考虑**
   - 限制上传文件类型
   - 限制文件大小
   - 文件名随机化（已实现）
   - 防止目录遍历攻击
